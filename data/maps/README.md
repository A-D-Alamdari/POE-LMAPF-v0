# Maps for the POE-LMAPF revision

This directory holds all MovingAI-format ``.map`` files used by the
simulator and the experimental harness.  The three maps required for
the paper Section 5.2 evaluation are:

| Stem                       | Dimensions (W x H) | Purpose                              |
| -------------------------- | ------------------ | ------------------------------------ |
| ``random-64-64-10``        | 64 x 64            | Random 10%-obstacle grid (open)      |
| ``warehouse-10-20-10-2-1`` | 161 x 63           | Warehouse with narrow service aisles |
| ``warehouse-10-20-10-2-2`` | 170 x 84           | Larger warehouse variant             |

All three are part of the standard MovingAI MAPF benchmark set.  They
are already checked in to this repository.  If you need to redownload
them (e.g. after a clean checkout, or to verify integrity), run
``scripts/download_maps.sh`` from the repo root.

## Canonical MovingAI URLs

The maps live in the MAPF benchmark archive at
<https://movingai.com/benchmarks/mapf.html>.  Direct file URLs:

- ``random-64-64-10.map``
  <https://movingai.com/benchmarks/random/random-64-64-10.map>
- ``warehouse-10-20-10-2-1.map``
  <https://movingai.com/benchmarks/warehouse/warehouse-10-20-10-2-1.map>
- ``warehouse-10-20-10-2-2.map``
  <https://movingai.com/benchmarks/warehouse/warehouse-10-20-10-2-2.map>

The MovingAI site also publishes ``.scen`` scenario files alongside the
maps.  The simulator does not require those for the paper Section 5.2
sweep — start/goal pairs are generated by the task allocator at run
time.

## Verifying maps after download

```bash
python -c "
from ha_lmapf.simulation.environment import Environment
for stem, expected in [
    ('random-64-64-10',        (64, 64)),
    ('warehouse-10-20-10-2-1', (161, 63)),
    ('warehouse-10-20-10-2-2', (170, 84)),
]:
    e = Environment.load_from_map(f'data/maps/{stem}.map')
    assert (e.width, e.height) == expected, (stem, e.width, e.height)
    print(f'{stem}: OK ({e.width}x{e.height}, {len(e._free_cells)} free cells)')
"
```

The same sanity check is exercised automatically by
``tests/test_map_loading.py``.
